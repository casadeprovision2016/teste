🎯 Características Otimizadas para seu Caso de Uso

  • Processamento assíncrono com Celery para não bloquear a API
  • Modelo Llama 3.2 3B quantizado para rodar eficientemente em container
  • Organização automática de arquivos seguindo padrão {ANO}/{UASG}/{PREGAO}
  • Sistema de callbacks para notificar conclusão do processamento
  • Cache inteligente de inferências para otimizar processamentos similares
  • Dashboard Flower para monitoramento em tempo real das filas
  • Auto-scaling de workers baseado no tamanho da fila
  • Retry automático com backoff exponencial em caso de falhas

🔗 Exemplo de Integração com API Externa

  • POST /api/v1/editais/processar - Recebe PDF e metadados (UASG, ano, número pregão)
  • Response: { "task_id": "uuid", "status": "queued" }
  • GET /api/v1/editais/status/{task_id} - Consulta status do processamento
  • GET /api/v1/editais/resultado/{task_id} - Obtém resultado estruturado
  • Webhook Callback: POST para URL configurada quando processamento concluir
